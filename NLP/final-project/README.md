# Pruning-RAG: Neural Network Pruning QA System

A Retrieval-Augmented Generation system for the neural network pruning + statistics domain.
Corpus: ~30 research papers (PDF). Supports hybrid retrieval, BGE reranking, and ablation experiments.

---

## Quick Start

### 1. Install dependencies
```bash
pip install -r requirements.txt
```

### 2. Add PDFs
```bash
cp your_papers/*.pdf data/pdfs/
```

### 3. Auto-build paper metadata
```bash
python scripts/build_papers.py
# → Writes data/papers.json. Review NEEDS_REVIEW entries.
```

### 4. Build the index (parse → chunk → embed → index)
```bash
python scripts/build_index.py
# Add --rebuild to reset the Chroma collection.
```

### 5. Ask a question
```bash
python scripts/ask.py "What is structured pruning?"
python scripts/ask.py --no-rag "What is structured pruning?"   # baseline
python scripts/ask.py --top-k 3 "Compare SNIP and GraSP."
```

---

## Evaluation

### Generate QA draft (requires index)
```bash
python eval/gen_qa_draft.py
# → eval/qa_set_draft.json   ⚠ MANUAL REVIEW REQUIRED before use
```
Edit `eval/qa_set_draft.json`, verify answers, then save as `eval/qa_set.json`.

### Retrieval metrics (Recall@k, MRR, nDCG@k)
```bash
python eval/eval_retrieval.py
```

### Generation metrics (RAGAS + citation accuracy)
```bash
python eval/eval_generation.py
python eval/eval_generation.py --no-rag   # No-RAG baseline
```

### Full ablation matrix → results.csv
```bash
python eval/run_experiments.py
python eval/run_experiments.py --only embedding   # one dimension
```

## Project Structure

```
pruning-rag/
  config.py          # All parameters — edit here, never hardcode elsewhere
  requirements.txt
  data/
    pdfs/            # ← Put your PDFs here
    parsed/          # Auto-generated by build_index.py
    papers.json      # Auto-generated by build_papers.py
    chroma/          # ChromaDB persistent store
    bm25.pkl         # BM25 index
  src/
    schemas.py       # Paper, Chunk pydantic models
    metadata.py      # Stage 0: title extraction + S2 API + topic classification
    parse.py         # Stage 1: Docling PDF → sections
    chunk.py         # Stage 2: section-aware token chunking
    embed.py         # Stage 3: BGE-M3 / MiniLM / Qwen3 embedders
    index.py         # Stage 4: Chroma + BM25 indexing
    retrieve.py      # Stage 5: dense + BM25 + RRF + reranker
    prompt.py        # Stage 6: chat message assembly
    generate.py      # Stage 7: Ollama / HF generation
    rag.py           # Stage 8: end-to-end orchestration
  eval/
    gen_qa_draft.py      # Auto-generate QA draft (needs human review)
    qa_set.json          # Gold QA set (human-verified)
    eval_retrieval.py    # Recall@k / MRR / nDCG@k
    eval_generation.py   # RAGAS + citation accuracy
    run_experiments.py   # Ablation matrix → results.csv
  scripts/
    build_papers.py      # Stage 0 CLI
    build_index.py       # Stages 1-4 CLI
    ask.py               # Query CLI
```

## Key Config Parameters (`config.py`)

| Parameter | Default | Description |
|---|---|---|
| `EMBED_MODEL` | `BAAI/bge-m3` | Embedding model |
| `CHUNK_SIZE` | `512` | Tokens per chunk |
| `CHUNK_OVERLAP` | `64` | Overlap between chunks |
| `SECTION_AWARE` | `True` | Section-aware chunking |
| `TOP_N` | `20` | Dense/sparse candidates each |
| `TOP_K` | `5` | Final chunks to LLM |
| `USE_HYBRID` | `True` | BM25 + dense fusion |
| `USE_RERANKER` | `True` | BGE cross-encoder reranker |
| `LLM_BACKEND` | `"ollama"` | `"ollama"` or `"hf"` |
| `LLM_MODEL` | `"llama3.1:8b-instruct"` | Ollama model name |

## Prerequisites

- Ollama running locally: `ollama pull llama3.1:8b-instruct`
- ~10 GB disk for models (BGE-M3 ~2.3 GB, reranker ~1.2 GB, LLM ~5 GB)
- Python 3.10+
