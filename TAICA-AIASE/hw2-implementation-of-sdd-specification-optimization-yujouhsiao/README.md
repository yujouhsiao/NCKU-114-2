# Balance Life CLI - 開發者設計與演化報告

## 1. 專案簡介
**Balance Life CLI** 是一個致力於平衡飲食花費與運動數據的命令行工具。
* **v1.0 階段**：建立核心的 CRUD（增刪查改）流程，使用 JSON 作為輕量級存儲，專注於基礎數據的持久化。
* **v2.0 演化**：轉向 **SQLite** 以處理更複雜的聚合查詢（Aggregation），引入「個人化目標系統」與「數據看板（Dashboard）」概念，協助使用者從單純的紀錄轉向決策分析。

---

## 2. v1.0 設計決策 (Design Decisions)
在開發 v1.0 時，我預先做了以下考量，以確保系統具備面對未來需求的擴充性：

* **三層架構 (3-Tier Architecture)**：
    * **決策**：將資料存取邏輯完全封裝在 `storage.py` 中。
    * **理由**：預判未來若資料量增加，純文本 JSON 會遇到效能瓶頸。透過分離存取層，我在 v2.0 更換儲存媒介時，**完全不需更動 `main.py` 的調用代碼**。
* **使用 UUID 作為唯一識別碼**：
    * **決策**：不使用簡單的數字遞增，而是採用 UUID 前 6 碼。
    * **理由**：考慮到未來若有「多裝置同步」或「資料匯入/合併」需求，UUID 能有效避免主鍵碰撞（Primary Key Collision）。
* **指令集的 Subparser 路由設計**：
    * **決策**：採用 `argparse` 的子指令模式（add, list, stats）。
    * **理由**：這讓 CLI 具備高度可擴充性。當 v2.0 需要新增 `goal` 指令時，我只需增加一個子解析器，而不必修改現有的參數結構，確保了介面的整潔與向下相容。

---

## 3. v2.0 實作說明
* **需求映射**：v2.0 要求將「靜態數據」轉為「動態回饋」。我將 PRD 中的「健康目標」映射為一個全新的資料表 `goals`，並在 `stats` 邏輯中進行跨表計算。
* **分類摘要演算法**：針對 `list` 的分組需求，我實作了動態聚合邏輯。當使用者查看今日紀錄時，程式會自動將紀錄按類別分類並計算小計，而非單純列印，這大幅提升了資訊的閱讀權重。
* **SQLite 參數化查詢**：為了防止 SQL 注入並提升通用性，所有查詢均採用參數化設計，確保系統在處理包含特殊字元（如：中文名稱）的輸入時依然穩定。

---

## 4. 向下相容性實作細節
* **自動遷移策略 (Migration Strategy)**：
    * 在 `storage.py` 初始化時，會自動檢查目錄下是否存在 `data.json`。
    * 若存在，系統會自動解析並將其轉換為 SQLite 紀錄，隨後將原檔案更名為 `.bak` 備份。這讓使用者可以無痛從 v1.0 切換至 v2.0。
* **介面一致性**：
    * v2.0 保留了 v1.0 定義的所有參數與退出碼。當執行舊有的測試案例（如負數報錯）時，輸出行為與 v1.0 完全一致，通過所有舊版測試。

## 5. 架構演化比較

| 面向 | v1.0 (基礎版) | v2.0 (演化版) |
| :--- | :--- | :--- |
| **儲存層** | 扁平 JSON 檔案 | **SQLite 關聯式資料庫** |
| **查詢能力** | 全量讀取後在記憶體篩選 | **SQL 條件過濾與 Group By 聚合** |
| **功能範疇** | 基本紀錄 | **目標對比、分類摘要、狀態提醒** |
| **模組化** | 初步分層 | **高度抽象化 (DAO Pattern)** |

## 6. 環境需求與執行方式

### 執行環境
* Python 3.7+
* 內建函式庫：`sqlite3`, `argparse`, `json`, `uuid`

### 執行範例 (v2.0)

```powershell
# 設定目標
python v2/main.py goal --budget 500 --workout 60

# 查看今日摘要看板 (自動分類小計)
python v2/main.py list

# 查看統計與目標達成狀況
python v2/main.py stats
```

## 7. 已知限制與未來改進方向 (v3.0 展望)

雖然 v2.0 已經大幅提升了實用性，但在實際壓力測試與數據分析後，仍具備以下優化空間：

### ⚠️ 目前的系統限制
* **週期性規劃能力不足**：目前的目標設定（Goal）僅支援「每日」維度，無法處理如「每月固定預算」或「每週運動總量」等較長週期的生活指標。
* **數據互動性**：目前僅提供命令行表格輸出，對於數據趨勢的掌握仍依賴使用者的數值感官，缺乏直觀的視覺反饋。

### 🚀 v3.0 預期改進方向
1. **數據視覺化 (Data Visualization)**：
    * 整合 `matplotlib` 函式庫，實作 `visualize` 指令。
    * 自動繪製「週開銷趨勢圖」與「運動時長圓餅圖」，讓數據分析更直觀。
2. **雲端異地同步 (Cloud Sync)**：
    * 支援將 SQLite 數據庫加密並同步至雲端伺服器（如 Firebase 或 AWS S3）。
    * 解決目前資料僅能存在於單機環境的限制，達成跨裝置數據一致性。
3. **多格式匯出功能 (Exporting)**：
    * 支援將活動紀錄導出為 `CSV` 或 `Excel` (.xlsx) 格式。
    * 便於使用者將資料導入 Google Sheets 或其他專業統計軟體進行深度二創分析。
4. **週期性預算管理**：
    * 擴充資料表結構，支援週、月、年等多維度的預算設定與提醒。
